# quota

Per-organization quota tracking and enforcement for the pipeline-builder platform. Stores each org's tier, limits, and usage counters, and exposes the increment/decrement primitives other services use to reserve and release quota. The API gateway (nginx) routes `/api/quota/*` here, rewriting to the service's internal `/quotas/*` paths.

## Responsibilities

- Holds per-org quota documents (limits + usage per type) keyed by `orgId`, seeded from tier presets (Mongo-backed).
- Answers quota lookups for the org's own quotas, a specific org (with authorization), and a single quota type.
- Provides system-admin views: all orgs with quotas, and an "at-risk" scan of orgs over a usage threshold (memoized).
- Supports system-admin limit/name/slug updates, usage resets, and a cascade-delete hook; offers internal-only `increment`/`decrement` endpoints for the reserve-then-rollback pattern used by gated actions.
- Quota types are sourced from `@pipeline-builder/api-core` (e.g. `plugins`, `pipelines`, `apiCalls`, `aiCalls`, `storageBytes`, `dashboards`, `alertRules`, `alertDestinations`, `idpConfigs`).

## Endpoints

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/quotas` | The caller's own org quotas (orgId from JWT) |
| GET | `/quotas/all` | All orgs with quotas (system admin) |
| GET | `/quotas/at-risk` | Orgs at/above a usage threshold, default 80% (system admin) |
| GET | `/quotas/:orgId` | All quotas for a specific org (authorized) |
| GET | `/quotas/:orgId/:quotaType` | Status for a single quota type (authorized) |
| PUT | `/quotas/:orgId` | Update name / slug / limits (system admin) |
| DELETE | `/quotas/:orgId` | Cascade-delete an org's quota doc, idempotent (system admin) |
| POST | `/quotas/:orgId/reset` | Reset usage counters (system admin) |
| POST | `/quotas/:orgId/increment` | Reserve usage (internal service-to-service) |
| POST | `/quotas/:orgId/decrement` | Roll back a reserved increment (internal service-to-service) |

## Development

Builds on the shared core packages (`@pipeline-builder/api-core`, `@pipeline-builder/api-server`, `@pipeline-builder/pipeline-core`). Project files are generated by projen from `.projenrc.ts`.

```sh
pnpm build   # compile + lint + test
pnpm test    # run tests
pnpm start   # run the compiled service (node lib/index.js)
```
