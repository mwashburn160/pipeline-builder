# reporting

Analytics and reporting for pipeline executions and plugin inventory. Ingests CI/CD lifecycle events and serves aggregated, per-organization dashboards. The API gateway (nginx) routes `/api/reports/*` here.

## Responsibilities

- Batch-ingests pipeline, stage, action, and plugin-build lifecycle events from a Lambda service account; events for unregistered pipeline ids are dropped, and re-deliveries are deduped.
- Serves execution analytics: per-pipeline counts, success rate over time, duration percentiles, and stage/action failure breakdowns.
- Serves plugin analytics: inventory summary, type/compute distribution, version counts, and build success/duration/failure metrics.
- Scopes all report queries to the caller's `orgId`; admin-only error/failure endpoints scrub credential-shaped substrings from messages.
- Caches aggregations in-memory (per org) and invalidates affected orgs after event ingest. Postgres-backed via the shared core packages.

## Endpoints

`/reports/events` requires auth but no org (a service account ingests across all tenants). The query routes require auth + org.

| Method | Path | Purpose |
|--------|------|---------|
| POST | `/reports/events` | Batch-ingest lifecycle events; returns `{ inserted, skipped, total }` |
| GET | `/reports/execution/count` | Execution count per pipeline with status breakdown |
| GET | `/reports/execution/success-rate` | Success rate over time (`interval`: day/week/month) |
| GET | `/reports/execution/duration` | Avg/min/max/p95 duration per pipeline |
| GET | `/reports/execution/stage-failures` | Stage failure heatmap |
| GET | `/reports/execution/stage-bottlenecks` | Slowest stages per pipeline |
| GET | `/reports/execution/action-failures` | Action (plugin step) failure rates |
| GET | `/reports/execution/errors` | Grouped error patterns (system admin; scrubbed) |
| GET | `/reports/plugins/summary` | Total/active/inactive/public/private/unique-name counts |
| GET | `/reports/plugins/distribution` | Plugin type Ă— compute type distribution |
| GET | `/reports/plugins/versions` | Version counts per plugin name |
| GET | `/reports/plugins/build-success-rate` | Build success rate over time (`interval`: day/week/month) |
| GET | `/reports/plugins/build-duration` | Avg/max build duration per plugin |
| GET | `/reports/plugins/build-failures` | Top build error messages (system admin; scrubbed) |

Time-ranged endpoints accept `from`/`to`; admin error/failure endpoints accept `limit`.

## Configuration

| Variable | Purpose | Default |
|----------|---------|---------|
| `PORT` | HTTP listen port | `3000` |
| `CACHE_TTL_REPORT_INVENTORY` | Inventory report cache TTL (seconds) | `300` |
| `CACHE_TTL_REPORT_TIMESERIES` | Timeseries report cache TTL (seconds) | `120` |
| `MAX_EVENTS_PER_BATCH` | Max events accepted per `POST /reports/events` | `100` |
| `PLATFORM_BASE_URL` | Platform base URL used for service-to-service calls | (core default) |

`CACHE_TTL_*` and `MAX_EVENTS_PER_BATCH` are read by `@pipeline-builder/pipeline-core` (`CoreConstants`); Postgres and JWT settings come from the shared core packages.

## Development

Builds on the shared core packages (`@pipeline-builder/api-core`, `@pipeline-builder/api-server`, `@pipeline-builder/pipeline-core`). Project files are generated by projen from `.projenrc.ts`.

```sh
pnpm build   # compile + lint + test
pnpm test    # run tests
pnpm start   # run the compiled service (node lib/index.js)
```
